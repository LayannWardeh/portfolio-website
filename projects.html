Project Overview

Developed a robust Unix system monitoring tool that continuously tracks and displays directory contents in real-time. This project demonstrated mastery of system programming, error handling, and user interface design in a terminal environment.

Technical Implementation

  • Built using pure C with extensive use of Unix system calls including opendir, readdir, stat, lstat, and readlink
  • Implemented automatic 3-second refresh cycle for real-time monitoring of file system changes
  • Created dynamic terminal UI that adapts to window resizing using TIOCGWINSZ ioctl calls
  • Developed comprehensive error handling for 15+ different failure scenarios
  • Replaced non-printable characters with placeholders to maintain display integrity

Key Features

  • Smart Display Management: Automatically adjusts column widths based on terminal size
  • File Content Preview: Shows first line of files, symlink targets, and directory indicators
  • Graceful Degradation: Displays "(truncated)" when content exceeds screen space
  • Window Size Detection: Shows "Window dimensions are too small" message when terminal is resized below minimum threshold
  • Comprehensive File Info: Displays file type, size, permissions (octal), owner name, and content summary

Error Handling Excellence

Achieved perfect score by implementing bulletproof error handling:

  • No segmentation faults possible under any invocation scenario
  • Graceful handling of permission denied, missing files, and invalid paths
  • Informative error messages using strerror() for all failure conditions
  • Continuous operation despite individual file access failures
  • Prevention of busy-waiting through proper delay implementation

Technical Achievements

  • Successfully handled edge cases including symbolic links, empty files, and special characters
  • Implemented efficient directory traversal without memory leaks
  • Created professional columnar output formatting with proper alignment
